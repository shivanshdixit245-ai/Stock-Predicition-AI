# Backtesting Engine — Full Specification

## Overview

The backtesting engine simulates trading on historical data using signals generated by the trained model. The critical differentiator vs. most student projects: realistic transaction costs, multiple benchmarks, and bootstrap significance testing.

**Interview talking point:** "My backtester includes a slippage model and confirms statistical significance of alpha via bootstrap permutation testing — without that, any backtest result could just be luck."

### Position logic
- HOLD → maintain current position
- BUY → enter long position (if not already long)
- SELL → exit long / enter short (or just exit long, configurable)
- Transaction cost applied on every entry and exit

## 2. Performance Metrics

Compute all of the following. Report in dashboard and log to MLflow.

### Return metrics
| Metric | Formula | Target |
|--------|---------|--------|
| Total return | (final_equity / initial - 1) × 100 | > benchmark |
| Annualised return (CAGR) | (1 + total_return)^(1/years) - 1 | > 10% |
| Win rate | # profitable trades / total trades | > 50% |
| Profit factor | gross_profit / gross_loss | > 1.3 |

### Risk-adjusted metrics
| Metric | Formula | Target |
|--------|---------|--------|
| Sharpe ratio | (mean_daily_return - rfr) / std_daily_return × √252 | > 1.0 |
| Calmar ratio | CAGR / max_drawdown | > 0.5 |
| Sortino ratio | mean_return / downside_std × √252 | > 1.2 |
| Max drawdown | max((peak - trough) / peak) | < 20% |
| Max drawdown duration | longest period underwater (days) | < 180 days |

## 3. Benchmark Comparisons

Compare strategy against three baselines (not just buy-and-hold):

### Benchmark 1: Buy and Hold
- Buy on day 1, hold until day N
- No transaction costs
- Simple but often hard to beat

### Benchmark 2: Momentum baseline
- Buy when 20-day SMA crosses above 50-day SMA
- Sell when 20-day SMA crosses below 50-day SMA
- Applies same transaction cost model as strategy
- Tests whether ML model adds value over a simple rule-based approach

### Benchmark 3: Random entry
- Random BUY/SELL/HOLD signals
- Run 1,000 random simulations
- Report mean and distribution of Sharpe ratios
- Used as null distribution for bootstrap significance test

## 4. Bootstrap Permutation Test (Statistical Significance)

This is the most important part of backtesting that almost no student project includes.

### The question it answers
Is our strategy's Sharpe ratio statistically different from what you'd get by chance?

### Method
```python
import numpy as np

def bootstrap_significance_test(
    strategy_returns: np.ndarray,
    n_permutations: int = 10_000,
    alpha: float = 0.05
) -> dict:
    """
    Permutation test: shuffle returns to destroy temporal structure,
    compute Sharpe under null hypothesis of no temporal pattern.
    """
    observed_sharpe = compute_sharpe(strategy_returns)
    
    null_sharpes = []
    for _ in range(n_permutations):
        shuffled = np.random.permutation(strategy_returns)
        null_sharpes.append(compute_sharpe(shuffled))
    
    null_sharpes = np.array(null_sharpes)
    p_value = np.mean(null_sharpes >= observed_sharpe)
    
    return {
        "observed_sharpe": observed_sharpe,
        "null_sharpe_mean": null_sharpes.mean(),
        "null_sharpe_std": null_sharpes.std(),
        "null_sharpe_95th_percentile": np.percentile(null_sharpes, 95),
        "p_value": p_value,
        "is_significant": p_value < alpha,
        "n_permutations": n_permutations,
    }
```

### Interpreting results
- p_value < 0.05 → strategy alpha is statistically significant at 95% confidence
- p_value > 0.05 → strategy may be overfitting; investigate further
- Always report p_value — do not hide it even if unfavourable

### Dashboard visualisation
Plot histogram of null distribution (grey) with observed Sharpe marked (vertical line, purple). Shade the 5% critical region. Show p-value annotation.

## 5. Regime-Conditional Backtest

Run separate backtest analysis per detected regime:

```
Bull regime:    Sharpe 1.8, win rate 62%, 48 trades
Bear regime:    Sharpe 0.3, win rate 44%, 22 trades  ← model weaker here
Sideways:       Sharpe 0.9, win rate 53%, 67 trades
```

This informs future model improvements and is a strong interview talking point: "I found the model has weaker signal in bear markets, which led me to investigate regime-specific features."
